# 二分查找

仅当列表有序才可用

在有序的列表中，从中间开始查找，比目标值大的在右边，小的在左边，每次从中间开始。

代码实现

def binary_search(list,item):
    low = 0 
    high = len(list)-1
  
    where low<=high:
        mid = (low + high)/2
	guess = list[mid]
	if guess == item:
	    return mid
	if guess > item:
	    high = mid -1
	else:
	    low = mid + 1
	return None

# 选择排序

遍历元素，每次抽出目标元素放在新的列表中，继续遍历，直到所有元素被处理。即有N给元素，要执行N遍选择。

时间复杂度：O(n*n)

并非每次都需要检查n个元素。第一次需要检查n个元素，但随后检查的元素数依次为n -1, n -2, …, 2和1。平均每次检查的元素数为1/2 ×n，因此运行时间为O(n×1/2×n)。但大O表示法省略诸如1/2这样的常数

代码实现

#元素从小按大排序

#找出给定数组中最小的元素
def findSmallest(arr):
    smallest = arr[0]
    smallest_index = 0
    #遍历当前数组每个元素
    for i in range(1,len(arr)):
        #选择最少的
        if arr[i] < smallest:
            smallest = arr[i]
            smallest_index = i
        return smallest_index;

def selectionSort(arr):
    newArry = []
    for i in range(len(arr)):
        smallest = findSmallest(arr);
        newArry.append(arr.pop(smallest))
    return newArry

# 分而治之（divide and conquer）

Q:如果有一小快土地，需要均匀的分成方块，且分出来的方块要尽可能的大，应该怎么分？

A: D&C策略 (1) 找出基线条件，这种条件必须尽可能简单。 (2) 不断将问题分解（或者说缩小规模），直到符合基线条件。

适合于边角料的最大方块，也适用于整个地的最大方块。

# 快速排序的思想

  • 对于排序来说，空数组或者已经排好序的数组是最简单的，不需要排序。
  • 因此基线条件是数组为空或只包含一个元素
  • 实际是二分查找的逆向操作,也是一种D&C思想的体现

代码实现

def quicksort(array):
    if len(array) < 2:
        return array;
    else:
        privot = array[0]
        # 小于等于基准值组成的子数组
        less = [ i for i in array[1:] if i<=privot]
        # 大于基准值组成的子数组
        greater = [ i for i in array[1:] if i>privot]
        return quckSort(less)+[privot]+qucksort(greater)

大O表示法

  1. 每层操作N个元素，因此每次O(n)
  2. 层高为log(n)
  3. 快速排序的时间复杂度为O(n*log(n))
  4. 最糟糕情况下每次作为基准值的元素都是最边缘，因此操作层高为N，所以复杂度为O(n*n)

散列函数：输入是一串信息（字符串），输出是一个数字 ❑ 散列函数很重要。前面的散列函数将所有的键都映射到一个位置，而最理想的情况是，散列函数将键均匀地映射到散列表的不同位置。 ❑ 如果散列表存储的链表很长，散列表的速度将急剧下降。然而，如果使用的散列函数很好，这些链表就不会很长！

# 广度优先搜索（无向图）

是一种图算法，解决最短路径问题的算法被称为广度优先搜索，帮助回答两种问题 ❑ 第一类问题：从节点A出发，有前往节点B的路径吗？ ❑ 第二类问题：从节点A出发，前往节点B的哪条路径最短？

# 工作原理

假设是要从人际关系网中查找芒果供应商

  1. 创建一个队列，储存要检查的人（直接关系）
  2. 依次从队列中弹出一个人，查看是否是芒果供应商
    • 如果是则完成
    • 如果否则将这个人的所有邻居加入到队列中，返回第二步
    • 如果队列为空，说明不存在

# 狄克斯特拉算法

  • 处理加权图，即对于需要带权重的最短路线问题,只适用于有向无环图（directed acyclic graph, DAG）。
  • 如果有负权边，就不能使用狄克斯特拉算法。因为负权边会导致这种算法不管用，这是因为狄克斯特拉算法这样假设：对于处理过的海报节点，没有前往该节点的更短路径。这种假设仅在没有负权边时才成立

# 工作原理

对于起点到终点的路线问题 (1) 找出“最便宜”的节点，即可在最短时间内到达的节点。 (2) 更新该节点的邻居的开销，其含义将稍后介绍。 (3) 重复这个过程，直到对图中的每个节点都这样做了。 (4) 计算最终路径。

代码实现

# 存储节点的邻居
graph["you"]=["alice", "bob", "claire"]

# 存储节点开销的散列表
infinity=float("inf")
costs={}
costs["a"]=6
costs["b"]=2
costs["fin"]=infinity

# 存储权重的散列表
graph["a"]={}
graph["a"]["fin"]=1

graph["b"]={}
graph["b"]["a"]=3
graph["b"]["fin"]=5

# 存储父节点的散列表
parents={}
parents["a"]="start"
parents["b"]="start"
parents["fin"]=None

node = find_lowest_cost_node(costs) # 在未处理节点中找出开销最小的节点
while node is not None:#所有节点都被处理过即结束
    cost = costs[node]
    neighbors = graph[node]
    for n in neighbors.keys():
        new_cost = cost + neighbors[n]
        if costs[n] > new_cost: # 如果当前节点前往该邻居更近，更新开销
            costs[n] = new_cost
            parents[n] = node # 同时该另据父节点设置为当前节点
    processed.append(node) # 标记为已处理过
    node = find_lowest_cost_node(costs) # 更新接下来需要处理的节点

def find_lowest_cost_node(costs):
    lowest_cost = float("inf")
    lowest_cost_node = None
    for node in costs: # 遍历所有节点
        cost = costs[node]
        if cost < lowest_cost and node not in processed: # 如果当前节点没有处理过且开销更低
            lowest_cost = cost # 将其视为开销最低的节点
            lowest_cost_node = node
    return lowest_cost_node

# 近似算法（approximation algorithm）

需要找出前往这5个城市的最优秀解，因此，需要计算媒体可能的路径

  • 旅行商问题和集合覆盖问题有一些共同之处：你需要计算所有的解，并从中选出最小/最短的那个。这两个问题都属于NP完全问题(即多项式复杂程度的非确定性问题)。
  • 如果能识别出问题属于NP问题，那么不用去寻找完美方案，而是使用近似算法即可。

示例 问题：假设你办了个广播节目，要让全美50个州的听众都收听得到，每个电台会覆盖多个州，存在有部分重叠，如何选择最小广播集合 (1) 选出这样一个广播台，即它覆盖了最多的未覆盖州。即便这个广播台覆盖了一些已覆盖的州，也没有关系。 (2) 重复第一步，直到覆盖了所有的州。

# 动态规划

# 背景问题

对于背包问题，背包可承重固定，有不同商品，商品重量价格各不同，选最佳组合。在不考虑条件限制情况下，一共会有2^n种组合，这样的话时间复杂度为O(2^n)

# 工作原理

D&C的思想，算出最小不可分情况下的最佳解，在前者基础上算出更广阔情况下的最优解。即解决大问题时，对于分解后剩下的小问题，采用小问题的最优解来解决。

对于背包问题

动态规划无法处理背包问题只偷商品一部分的情况，贪婪算法可以，优先拿价值高，然后次之。

  • 动态规划功能强大，它能够解决子问题并使用这些答案来解决大问题。但仅当每个子问题都是离散的，即不依赖于其他子问题时，动态规划才管用。这意味着使用动态规划算法解决不了去巴黎玩的问题。
  • 每种动态规划解决方案都涉及网格。
  • 单元格中的值通常就是你要优化的值。在前面的背包问题中，单元格的值为商品的价值
  • 每个单元格都是一个子问题，因此你应考虑如何将问题分成子问题，这有助于你找出网格的坐标轴

# K最近邻算法

K最近邻（k-nearest neighbours, KNN，通过特征抽取，比对K个特征相似度，来归类（或描述相似程度）。

主要涉及以下两项工作 ❑ 分类就是编组； ❑ 回归就是预测结果（如一个数字）。

计算两位用户的距离时，使用的都是距离公式。还有更合适的公式吗？ 在实际工作中，经常使用余弦相似度（cosine similarity）,余弦相似度不计算两个矢量的距离，而比较它们的角度。但如果你要使用KNN，就一定要研究研究它！
