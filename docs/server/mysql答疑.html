    # Mysql答疑

    # mysql索引实现

    1. mysql索引实现：
      1. hash：哈希表，通过key的哈希值找到value，很快，但不支持范围查询和排序查询；时间复杂度O(1),排序会退化为O(N),树形结构永远是O(log(n))
      2. b-tree：自平衡的多路搜索树，b代表balance，比起二叉树，树高度低，IO次数小

    # b-tree和b+tree

    B+树和B树的区别 - 简书 (jianshu.com) (opens new window)

    1. b树
      1. 所有结点和叶节点都存储索引和data数据
      2. 搜索可能在一个节点结束，最低复杂度为O(1)
      3. 任何一个关键字出现且只出现在一个结点
    2. b+树
      1. 所有关键字在叶子结点内出现，结点只存储关键字，叶节点存储索引和data数据，复杂度为固定log(n)
      2. 为所有叶子节点添加一个链指针，大大增加区间访问性，利用了空间局部性原理
      3. 内结点没有data，所以单次IO信息量比b-tree大

    # mysql的引擎

    1. myISAM：
      1. b+tree叶节点存放数据地址，先根据索引找到对应的地址，再根据地址读取数据，成为非聚集索引
    2. innoDB:
      1. 主键索引是聚集索引，主键和行记录放在同一个叶节点，找到主键即找到行记录；
      2. 非主键索引都是非聚集索引，innoDB的非聚集索引的索引文件和数据文件分开存储，叶节点保存的是主键，所以还需要根据主键去索引文件找详细行记录。主键字段不宜过长

    # 回表查询

    在innoDB引擎中，非主键索引查找数据需要先找到主键，再根据主键查找具体行数据，就叫回表查询
