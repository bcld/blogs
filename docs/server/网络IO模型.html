# 网络IO模型

网络io模型是用来接受和发送数据的通道模型。JAVA共支持NIO\BIO\AIO三种网络IO模型

Netty学习前基本知识 — BIO 、NIO 、AIO 总结 (qq.com) (opens new window)

# 基础概念

怎样理解阻塞非阻塞与同步异步的区别？ (opens new window)

  • 同步：发起调用后，被调用者未处理完请求前，不返回调用。
  • 异步：发起调用后，立刻得到表示已收到调用的返回（非调用结果）。被调用者继续处理，通常依赖回调机制来通知调用者其执行结果。
  • 阻塞：发起请求后，当前线程等待调用结果返回。
  • 非阻塞：发起请求后，调用方不用一致等待结果返回，先去执行别的事情。

两者其实很类似，只是描述角度不同，前者更描述的是交互行为，后面描述的是自己的状态

所有系统IO都分为2部分，等到就绪（不适用cpu） 和 操作

# BIO （blocking I/O）

同步阻塞I/O，典型的 一请求一应答通信模型 。

  1. 独立的 Acceptor 线程负责监听客户端的连接，我们一般通过在 while(true) 循环中服务端会调用 accept() 方法等待客户端连接的方式监听请求，请求一旦接收到一个连接请求，就可以建立通信套接字在这个通信套接字上进行读写操作，此时不能再接收其他客户端连接请求，只能等待当前连接的客户端的操作执行完成，不过可以通过多线程来支持多个客户端的连接，如上图所示。
  2. 要让BIO通信模型同时处理多个客户端请求就要多线程。（原因是 socket.accept()、 socket.read()、 socket.write() 涉及的三个主要函数都是同步阻塞的）

使用线程池可以在有效控制内提高cpu利用率，实现伪异步I/O模型

# 线程是宝贵的资源

  • 线程的创建和销毁成本很高，在linux中是重量级系统函数
  • 线程本身占用较大内存，java线程至少分配512k~1M
  • 线程切换成本高，频繁可能会使线程切换时间比执行时间长

# 伪异步I/O

采用线程池+任务队列实现

缺点：

  • read操作时阻塞操作，如果连接不做数据读写会则导致线程阻塞，浪费资源
  • 线程池的线程太多，会对服务器造成压力

# NIO(Non Blocking IO)

Java NIO浅析-美团技术团队 (opens new window)

同步非阻塞，服务器实现模式为一个线程可以处理多个请求（连接），客户端发送的连接请求都会注册到 多路复用器 selector 上，多路复用器轮询到连接有 IO 请求就进行处理。

它支持面向缓冲的，基于通道的I/O操作方法。NIO提供了与传统BIO模型中的 Socket 和 ServerSocket 相对应的 SocketChannel 和 ServerSocketChannel 两种不同的套接字通道实现，两种通道都支持阻塞和非阻塞两种模式。

# NIO核心组件

  • channel（通道），双向，可读也可以写，而流是单向的
  • buffer（缓冲区），在NIO中所有数据都在缓冲区中处理
  • selector（选择器）

BIO：后端线程需要阻塞等待客户端写数据,如果客户端不写数据就阻塞

NIO：selector负责轮询所有已注册的客户端，发现有事件发生才转给后端线程处理，后端线程不需要阻塞等待，且channel的读写非阻塞，通过这种方式，甚至单线程都能做到高效的复用，最典型的例子就是Redis了。

# 选择器

NIO浅谈代码 (opens new window) 选择器作用就是 某一状态已就绪时触发线程处理。selector的实现方案：

  • select，存在有连接准备就绪时，能够知道但是不能知道具体是哪一个来连接，需要线性遍历所有连接，时间复杂度为O(n),有最大链接数限制
  • poll，底层采用链表，理论没有最大连接数限制
  • epoll，事件通知方式，内核实现中epoll是根据每个fd（文件描述符）上面的callback函数实现，只要就绪就会回调callback函数，精准通知。事件复杂度为O（1），目前只有linux支持

# AIO（Asynchronous I/O）

是异步非阻塞的IO模型，异步 IO 是基于事件和回调机制实现的，也就是应用操作之后会直接返回，不会堵塞在那里，当后台处理完成，操作系统会通知相应的线程进行后续的操作。

适合于连接数目多且连接比较长（重操作）的架构

BIO NIO AIO
IO模型 同步阻塞 同步非阻塞（多路复用） 异步非阻塞
编程难度 简单 复杂 复杂
可靠性 差 好 好
吞吐量 低 高 高

# 答疑

  1. IO是什么 数据通过媒介读取和写入都是IO，网络IO流程：网卡->内核空间->用户空间->应用程序

  2. NIO是不是就是I/O多路复用? 以下是多路复用的流程，网卡还没准备好写入到内核空间时候，直接返回程序未就绪，该线程继续别的事情，直到数据准备好，通知对应的请求进行IO（IO过程是阻塞的），其他请求等待是同步的，但是程序处理请求时非阻塞的.

    NIO相当于多路复用的必要条件。
